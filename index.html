Welcome to the WoLERY

  • W3C has produced specs.
  • OWL and RDF working groups have disolved
  • Can't fix errors without chartering a WG.

W3C has produced a number of documents that provide specifications and other information relating to OWL and RDF, under the auspices of carefully and narrowly chartered working groups.

Unfortunately, once a working group has finished the tasks assigned described in its charter, it is disbanded. If errors and ommisions are discovered in a recommendation, even if they are purely typographical, there is no way to emend the recommendations without rechartering a new working group.

WHO-WG: Mission Statement

If I am not for myself, WHO will be.
  • Prepare corrected versions of relevant OWL and RDF recommendations, incorporating known errata.
  • Rough consensus and running code.
  • Use version control (i.e. GIT) to track changes to base spec.
  • If possible, ask editors of documents being modified to review or edit any proposed changes.
  • If possible, use prior WG mailing lists for discussion.
  • If possible, track issues using numbers that do not conflict with relevant WGs ISSUE-XX numbering
  • Ensure modified documents are clearly distinguished from published recommendations (e.g. do not use W3C logo, use distinct document status.
  • Where not in conflict with above, follow W3 style guide.
  • Separate corrections and clarifications (levels 1-3) from new features (level 4)
  • Produce documents that can readily be taken up by appropriately chartered W3 recommendation track working group.
